# Expansion Method — node N2

Reference for `N2-discover`. SKILL.md holds the *workflow*; this file explains
the *method*: why Semrush variations/questions are the engine, the seed gate, the
relevance filter, the output schema, the Phase-2 multilingual pass, and the free
Autocomplete fallback.

## Why Semrush variations/questions (manual phase)

Two ways to expand a seed:

- **Free Autocomplete** returns strings real users type — grounded, but with **no
  volume**. You learn *that* a phrase is searched, not *how much*. Building
  priority on top of it means guessing demand.
- **Semrush "Keyword Variations" + "Questions"** returns, for each seed,
  thousands of related queries and the questions people ask — **each with real
  monthly volume and KD**. (Observed for ChinaReady: a single seed like
  `vpn for china` surfaces ~22K variation terms and ~2.9K questions, each scored.)

So in the manual phase the engine is Semrush. Every harvested row arrives already
carrying the metric N3 needs, which is why paid data enters the chain **here at
discovery** rather than as a fragile keyword-match merge later. The LLM's job is
not to invent phrases; it is to gate seeds, relevance-filter the harvest, and
cluster it.

## The seed gate (before expansion)

N1 hands over 10–20 seed roots. Not all deserve expansion. Run them through
Semrush Keyword Overview (data point A) and keep a seed if **either**:

1. **Volume ≥ ~100** — there's real English demand at the root, **or**
2. **Strategic binding** — the seed maps to a core tool / flagship page (a free
   checker, the pillar guide). Keep these **even at low root volume.** Example:
   `esim or vpn for china` shows 0 root volume, but it's the exact job of the
   `/esim-vpn-checker` tool, and its variations/questions
   ("esim or vpn for china reddit", "do i need both esim and vpn in china")
   carry the demand. Dropping the highest-converting tool clusters because the
   *root* is thin would be a mistake.

Everything else is parked (noted, not deleted from N1). This is the single
biggest noise-control lever: expanding a dead root multiplies dead long-tails,
and translating a dead root (Phase 2) multiplies them across languages.

Worked example (ChinaReady seed overview):

| seed | volume | keep? | why |
|---|---|---|---|
| `vpn for china` | 6600 | ✅ | strong demand |
| `china vpn` | 5400 | ✅ | strong demand |
| `china esim` | 1900 | ✅ | demand |
| `china visa free` | 1300 | ✅ | demand |
| `google maps china` | 720 | ✅ | demand |
| `wechat pay for foreigners` | 260 | ✅ | demand |
| `china 240 hour transit visa` | 110 | ✅ | demand + tool page |
| `esim or vpn for china` | 0 | ✅ | **strategic** → `/esim-vpn-checker` |
| `alipay for tourists` | 20 | ✅ | **strategic** → payments guide |
| `china travel checklist` | 20 | ⚠️ | strategic (product name); low priority |
| `apps to download before china` | 10 | ➖ | park unless a guide needs it |
| `first time china travel` | 10 | ➖ | park |

## Ingest: aggregate → dedupe → gate

`ingest-semrush.py` turns the `_semrush/` folder into `keywords-raw.csv`:

- **Source from filename.** `vpn-for-china__variations.csv` → seed
  `vpn for china`, source `variation`; `__questions.csv` → source `question`;
  `00_seed-overview.csv` → source `overview`. The seed slug is matched (case/
  space/hyphen-insensitive) against the N1 profile's seeds (via `--profile`) to
  carry over `cluster`/`layer`.
- **Header mapping.** Accepts Chinese Semrush UI headers (`关键词 / 意图 / 搜索量 /
  KD (%) / CPC (USD)`) and English variants (`Keyword / Intent / Volume /
  Difficulty / CPC`), CSV or TSV (handles the tab-separated "复制" paste).
- **Intent prior.** The Semrush intent cell (`I`, `C`, `C, I`, or full words) is
  normalized to readable labels in `semrush_intent` — a *prior* for N3, not the
  final call.
- **Dedupe.** Across all files by `(lowercase keyword, lang)`; the richest record
  wins (max volume), a `question` source label is preferred, intents are unioned.
- **Volume floor.** `--min-volume N` drops long-tails below N. Blank-volume rows
  (e.g. overview seeds, or a question Semrush didn't score) are kept — absence of
  a number isn't evidence of no demand.

## Relevance filter (the LLM judgment step)

Semrush variations are generated by lexical/semantic proximity, so a travel seed
drags in high-volume but **off-topic** terms. Cut them against N1's
jobs-to-be-done. Examples to cut under ChinaReady seeds:

- `china japan travel warning` (8.1K) — news/geopolitics, not pre-arrival readiness
- `wells fargo china travel ban` (2.9K) — corporate policy, unrelated
- `china travel advisory` (2.4K) — safety advisory, not the product's job

Keep the ones that match a real user job, e.g.:

- `can americans travel to china` (1.3K, KD 17) — eligibility, maps to visa/transit
- `how to get travel visa for china` (1.0K) — maps to transit-checker / visa guide
- `do i need a visa to travel to china` (590) — same cluster

After cutting, verify/assign each kept row's `cluster` so N3 can group a topic.
This is N2's close: **aggregate → relevance-filter → dedupe → cluster.** Do *not*
assign intent/funnel/priority (N3) — but the kept set is already ordered by real
demand because volume rode in with every row.

## Output schema — keywords-raw.csv

| column | meaning |
|---|---|
| `keyword` | the query, exactly as Semrush exported it |
| `seed` | which seed produced it (from the filename / profile) |
| `layer` | `core` / `scenario` / `pain`, carried from the N1 seed |
| `source` | `overview` / `variation` / `question` |
| `country` | `gl` tag (default `us`) |
| `lang` | language tag (default `en`; the Phase-2 axis) |
| `cluster` | topic slug, from N1 or assigned in the relevance step |
| `volume` | Semrush monthly volume (the demand signal N3 scores on) |
| `kd` | Semrush Keyword Difficulty 0–100 (authoritative difficulty for N3) |
| `cpc` | Semrush CPC (commercial-value hint) |
| `semrush_intent` | Semrush's intent label(s) — a prior for N3's intent call |

Dedup is by exact `(lowercase keyword, lang)`.

## Phase 2 — multilingual (deferred, winners only)

The mistake to avoid: translating *all* N1 seeds up front. Several have little
English volume; translating them just multiplies noise across languages.

The right order:

1. Run **English** through N2 → N3, get the ranked set.
2. Take the **winners** — high volume, winnable KD, tool/comparison/question
   intent.
3. The LLM translates *those* into each target language's **idiomatic** phrasing
   (not literal): German `china esim mit nummer`, French `chine sans visa pour
   les français`. Phrasing, not word-for-word.
4. Pull Semrush for the translated terms (data points A/B again, per language),
   or use `expand-keywords.py` LOCALES mode as a free probe.
5. Ingest → N3, tagging `lang` per language.

Which languages: match the product's real source markets (for inbound-to-China
travel: German, French, Spanish, Korean, Japanese, Italian, …). Start with the
2–3 biggest and expand based on what scores.

The v1 all-seed multilingual harvest (`N2_keywords-raw.csv`, 1116 rows en/de/fr/
es, zero volume) is kept for reference but is **not** the current path — it
expanded every seed including dead ones, which is exactly what the gate now
prevents.

## Appendix — Autocomplete fallback (`expand-keywords.py`)

Free engine for when there's no Semrush data (zero budget) or to surface fresh
angles to add as new seeds. It harvests Google's public Autocomplete:

```
https://suggestqueries.google.com/complete/search?client=firefox&hl=en&gl=us&q=<query>
```

Each **seed** becomes many **probe queries**; the completions are the harvest
(the probe text itself is never output):

| source label | probe shape | catches |
|---|---|---|
| `base` | the seed as-is | headline long-tails |
| `question` | `do i need {s}`, `can i {s}`, `how to {s}`, `is {s}`, `what is {s}` … | PAA-style questions |
| `comparison` | `{s} vs`, `{s} or`, `best {s}`, `{s} alternative` … | decision/comparison queries |
| `modifier` | `{s} for tourists`, `{s} 2026`, `{s} app`, `{s} reddit` … | qualified long-tails |
| `alphabet` | `{s} a` … `{s} z` (only `--depth full`) | the wide net |

**Two localization axes** (LOCALES mode is the real one): translate seeds AND
templates into a language and probe under its `hl` to capture native phrasing
(`china esim für touristen`); `--gl us,gb` is only the weaker geo axis. See the
script header and `locales.json` shape. **Caveat for the fallback:** rows have
**no volume** — so N3 scores them on LLM difficulty, and they're best used to
discover angles you then validate with Semrush, not as a final list.

## Failure modes

- **`ingest-semrush.py` says "no Keyword column"** → the export's header isn't
  recognized. Check it has a `关键词`/`Keyword` column; re-export if Semrush wrote
  a metadata preamble above the header row.
- **A seed's rows all look off-topic** → the seed is ambiguous; the relevance
  filter should cut them, and consider gating that seed out next run.
- **Low total volume for a seed** → it probably should have been gated out at
  step 2; park it unless it's a strategic tool-page seed.
- **Autocomplete (fallback) returns empty for everything** → soft rate-limit or
  proxy issue. Raise `--sleep`, reduce seeds, retry. The script fails open
  (returns `[]`), so a bad run yields a thin CSV, not a crash.
